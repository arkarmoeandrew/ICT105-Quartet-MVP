# Business Model Canvas

## Project Title
RSU Nexus – Campus Resource Marketplace

## 1. Customer Segments
- Primary users: RSU students looking to buy, sell, or exchange second-hand items.
- Secondary users: Student clubs, organizations, and campus administrators who oversee marketplace activities.

## 2. Value Proposition
RSU Nexus provides a safe and organized campus marketplace where students can easily buy, sell, and exchange second-hand items. It reduces waste, helps students save money, and makes finding campus resources more convenient.

## 3. Channels
- Web application accessible through desktop and mobile browsers.
- University announcements and student orientation.
- Social media and student community groups.
- QR codes shared around the campus.

## 4. Customer Relationships
- User registration and login.
- User profiles and listing management.
- Notifications for listing updates and status changes.
- Private messaging between buyers and sellers.
- Feedback and ratings for completed transactions.

## 5. Revenue Streams / Sustainability Logic
The prototype is not intended to generate direct revenue. Its value comes from:
- Supporting sustainable reuse of campus resources.
- Reducing unnecessary purchasing costs for students.
- Improving campus resource sharing efficiency.
- Future opportunities may include promoted listings or university sponsorship.

## 6. Key Resources
- Development team members.
- HTML, CSS, JavaScript, and PHP.
- Supabase.
- GitHub repository.
- Claude Design for UI/UX Design.
- Web hosting and internet connection.

## 7. Key Activities
- Develop and maintain the web application.
- Manage user accounts and marketplace listings.
- Provide search and filtering functions.
- Monitor and moderate inappropriate listings.
- Maintain database and system performance.

## 8. Key Partners
- Rangsit University.
- Students and student organizations.
- Project supervisor/lecturer.
- Web hosting provider.
- GitHub for version control.

## 9. Cost Structure
- Development time.
- Internet and hosting services.
- Domain (if deployed publicly).
- Maintenance and bug fixes.
- Design and collaboration tools.

## Final Note
This business model directly supports the MVP features of RSU Nexus, including user registration, posting resource listings, browsing resources, searching and filtering, viewing item details, private messaging, status tracking, admin management, and dashboard summary. These features work together to provide a complete campus resource marketplace prototype that addresses the identified student needs.
