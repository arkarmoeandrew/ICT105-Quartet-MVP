## Wireframe Quality Rules

- Use consistent navigation, colors, buttons, and labels across all wireframes.
- Avoid screens that are only decorative and do not provide functionality.
- Display realistic sample resource listings instead of empty placeholders.
- Ensure every wireframe supports the corresponding system requirements.
- Wireframes may be created using Figma, Canva, PowerPoint, HTML/CSS, or another suitable prototyping tool.

The paths above match the files currently stored in the repository. The binary wireframes were not renamed during the Lab 01–08 text-consistency review.
