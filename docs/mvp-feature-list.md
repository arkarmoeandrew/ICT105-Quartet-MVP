# Lab 04 – MVP Feature List

## MVP Decision Rule
The MVP should include the smallest set of features that can demonstrate the core problem–solution fit by the end of the semester.

## MoSCoW Prioritization

| Feature ID | Feature Name | Problem Solved | Related User Story | Priority (Must/Should/Could/Won't) | Technical Complexity (1–5) | User Value (1–5) | Evidence Strength (1–5) | Include in Final Prototype? |
|------------|--------------|----------------|--------------------|------------------------------------|----------------------------|------------------|-------------------------|------------------------------|
| F-01 | Homepage and Verified Access | Explains the platform and limits access to verified RSU students | US-01 | Must | 2 | 5 | 5 | Yes |
| F-02 | Create Resource Listing | Allows students to post equipment, learning resources, or services | US-02 | Must | 3 | 5 | 5 | Yes |
| F-03 | Browse, Search, and Filter Marketplace | Helps students quickly find needed resources or services | US-03 | Must | 3 | 5 | 5 | Yes |
| F-04 | View Listing Details and Status | Lets students view complete information and availability before requesting | US-04 | Should | 2 | 4 | 4 | Yes |
| F-05 | Submit Request or Contact Provider | Allows students to request a resource or student service | US-05 | Must | 3 | 5 | 4 | Yes |
| F-06 | Manage My Listings | Allows users to edit, update, or delete their own listings | US-06 | Could | 3 | 4 | 3 | Optional |
| F-07 | Admin Listing Management | Allows an authorized administrator to review and manage listings | US-07 | Must | 3 | 4 | 3 | Yes |
| F-08 | Marketplace Dashboard | Shows summary information about listings, categories, and requests | US-08 | Should | 2 | 4 | 3 | Yes |
| F-09 | Complex Booking / Reservation | Adds scheduling and reservation logic beyond the request workflow | Future | Won't | 5 | 4 | 2 | No |

## Must-Have Features
- **Homepage and Verified Access (F-01)** – Explain the platform and verify users using RSU email authentication.
- **Create Resource Listing (F-02)** – Students can create listings for equipment, learning resources, or services.
- **Browse, Search, and Filter Marketplace (F-03)** – Browse records and search by keyword and category.
- **Submit Request or Contact Provider (F-05)** – Request resources or services from the listing provider.
- **Admin Listing Management (F-07)** – Allow authorized administrators to manage marketplace content.

## Should-Have Features
- **View Listing Details and Status (F-04)** – Display complete information and availability before a request.
- **Marketplace Dashboard (F-08)** – Display marketplace and request summary information.

## Could-Have / Future Features
- **Manage My Listings (F-06)** – Edit, update, or delete personal listings.

## Not in MVP
**Complex Booking / Reservation (F-09)** is intentionally excluded from the MVP because it has high technical complexity, weaker supporting evidence, and is not essential for demonstrating the core request-and-approval workflow. It can be added as a future enhancement after validating the MVP.
